### 4. 📊 Performance Metrics (Risk-Adjusted)
**Purpose**: Show judging criteria compliance

#### Sharpe Ratio (Risk)
- **What**: Return per unit of risk taken
- **Hackathon Goal**: > 1.0 is competitive
- **Why**: Judges prefer consistent returns over volatile ones
- **ERC-8004 Reward**: Risk-adjusted profitability = primary metric

#### Max Drawdown (Risk Control)
- **What**: Largest peak-to-trough decline
- **Hackathon Goal**: < 20% is excellent
- **Why**: Shows you manage catastrophic losses
- **ERC-8004 Reward**: Drawdown control = special award category

#### Total Return (PnL)
- **What**: Overall profitability
- **Hackathon Goal**: > 5% for demo is strong
- **Why**: Proves trading strategy works
- **Kraken Reward**: Pure PnL ranking (net profit)

**Judging Rubric These Impact**:
1. **Application of Technology** - Show you use metrics for decisions
2. **Business Value** - Returns prove viability
3. **Risk Management** - Demonstrate controls

## Troubleshooting

### NFT Not Showing?
1. Check AgentAddress in localStorage
2. Verify transaction on Sepolia
3. Try minting again
4. Check console for errors

### Sharpe Ratio = 0?
1. Need at least 5 trades
2. Check trades have entry/exit prices
3. Verify win_rate is calculated
4. Test with `GET /metrics` endpoint

### Leaderboard Shows N/A?
1. Check agent is registered at early.surge.xyz
2. Wait 5 minutes for indexing
3. Try refreshing page
4. Check Discord for leaderboard status

### Compliance Controls Not Enforcing?
1. Verify Risk Router contract is live
2. Check position sizes vs limits
3. Test with small trade first
4. Contact Discord support
